# Rule-Based Language with Execution and Analysis

Final project for the Formal Languages course. It implements a system that
defines, processes and executes a small rule-based language of the form
`if A then B`, following the classic compiler pipeline: lexical analysis,
parsing, AST construction, interpretation and a static analysis module.

## Project structure

- `lexer.py` — lexical analyzer (text to tokens)
- `ast_nodes.py` — AST node classes
- `parser.py` — LL(1) recursive descent parser
- `interpreter.py` — fixed-point evaluation of the rules
- `analyzer.py` — static analysis (conflicts, redundancy, inactive rules)
- `main.py` — entry point
- `tests/` — test case files
- `Data/` — project statement

## How to run

The program receives a single input file containing the rules and an optional
`State:` section:

```
python main.py tests/case1.txt
```

The program prints the activated facts (one per line, in alphabetical order, or
`(no output)` if there are none) and then the static analysis messages.

The `tests/` folder contains the 8 test cases from the project statement plus
the extra grading case.
